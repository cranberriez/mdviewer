# v0.4 Changelog

Changes accumulated on the `v0.4` branch through `0.4.8`.

## Added

- Added an Explorer filter menu so show/hide options live in the Explorer context menu
  instead of taking permanent sidebar space.
- Added **Open as Root** to folder context menus, allowing any visible folder to become the
  active project root.
- Added a refreshed app icon.
- Added 5-second auto-save for dirty open files.
- Added preview handling for YAML frontmatter so leading `---` metadata blocks no longer
  render as document noise.
- Added protected frontmatter display in Preview and visual Edit mode.
- Added a Recent sidebar source with up to five recently opened files for the active root,
  plus a nested File > Open Recent menu for the ten most recent roots and dashboard history.
- Added a titlebar Home action and a dashboard startup preference for choosing between Home
  and the previous workspace.
- Added default-off preferences for Windows Explorer integration. Markdown files and folders
  can be enabled independently, with single-instance routing into the existing app window.

## Changed

- Moved Explorer modes and workspace controls into a persistent vertical activity rail, with
  the navigation toggle at the bottom and a dashboard-style Home action in the title bar.
- The saved-location list now keeps the active root visible and exposes pin controls directly
  on hovered folder rows.
- Reworked saved-location menus, Explorer header actions, and menu-bar actions to use shared
  menu definitions for more consistent behavior.
- Improved the saved-location icon picker flow so changing an icon can keep the menu context
  open where needed.
- Refactored the sidebar, preview panel, and visual markdown editor into smaller focused
  pieces to make the app easier to maintain without changing the core workflow.
- Updated the app build stack, including Vite, Tauri, Tailwind/Vite tooling, GitHub Actions,
  and related dependencies.
- Saved markdown files now end with one trailing newline for cleaner editing and POSIX-friendly
  output.
- Switching roots now preserves the active view mode and reopens the last file used in that root.
- The setup Home folder picker now changes the actual Home location instead of only pinning the
  selected folder.

## Fixed

- Stabilized menu dismissal behavior so context menus and popovers close more reliably.
- Hardened outline anchor generation and visual markdown serialization against malformed or
  multi-character input.
- Fixed release, dependency-audit, and version-bump automation used to ship the app.
- Protected YAML frontmatter from visual-editor round trips so metadata blocks remain visible
  without being reformatted as normal markdown content.
- Removed the embedded browser context menu from app surfaces without a custom right-click menu.

## Maintenance

- Added GitHub release-build automation for merges to `main`.
- Added CodeQL, dependency audit, Dependabot grouping, and a project security policy.
